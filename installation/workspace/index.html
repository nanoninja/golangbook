Espace de Travail

  • Configurer un espace de travail

Un espace de travail (Workspace) est une hiérarchie de répertoires dédiée à la création d’applications et de paquetages.
Il est créé par défaut dans votre dossier personnel et se nomme go.

Cet espace doit avoir 3 dossiers de niveau supérieur appelés :

go
├── bin
├── pkg
└── src

  • bin contient les binaires exécutables provenant de dépendances tierces ou de la compilation de
    vos programmes via la commande go install

  • pkg ce répertoire est utilisé par Go et contient les dépendances des modules et des fichiers avec
    l’extension .a (archive) qui sont des objets de package Go compilés et utilisés au moment de la liaison pour
    créer le binaire exécutable complet dans le répertoire bin

    Il est possible de compiler un package une fois et lier cet objet à d’autre exécutables.
    Par exemple le package fmt qui apparaît dans presque tous les programmes est
    compilé une fois pour être lié plusieurs fois.

  • src contient les fichiers source Go

go
├── bin
├── pkg
│   ├── linux_amd64
│   │   └── hello.a
│   ├── mod
│   └── sumdb
└── src
    ├── github.com
    │   └── nanoninja
    │       └── main.go
    └── hello
        └── hello.go

Configurer un espace de travail

La configuration de l’espace de travail se fait en déclarant le chemin d’accès dans la variable d’environnement GOPATH.

Vous pouvez modifier la variable temporairement mais elle sera valable que si le terminal reste actif :

go env -w GOPATH=/tmp/workspace/go

Pour rendre le chemin d’accès de votre espace de travail persitent il faut l’enregistrer dans le système d’exploitation.

  • zsh : $HOME/.zshrc
  • bash : $HOME/.bashrc
  • windows : Panneau de configuration > Système et sécurité > Système > Paramètres système avancés
