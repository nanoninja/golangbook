Obtenir la version de Go

go version

Version de Go utilisé par un binaire :

go version -m [file ...]

Afficher l’aide

go help

go help build

Accéder à la documentation en ligne de commande

La documentation est alimentée par les commentaires rédigés par les développeurs dans le code source et est accessible en locale dans le terminal en ligne de commande :

go doc

Exemples :

Commande Description
go doc Affiche la documentation du package actuel
go doc Foo Affiche la documentation de Foo dans le package actuel
go doc encoding/json Affiche la documentation du package encoding/json
go doc json Raccourci pour encoding/json
go doc json.Number Afficher la documentation et le résumé de la méthode pour json.Number

Pour en savoir plus consultez l’aide de la commande doc :

go help doc

Accéder à la documentation depuis un serveur

Go propose d’installer un package pour obtenir la documentation au format web via un serveur :

go install golang.org/x/tools/cmd/godoc@latest

Par défaut le serveur est configuré pour fonctionner avec les modules mais il est possible de le démarrer en les désactivant temporairement avec la variable d’environnement GO111MODULE.

go env -w GO111MODULE=off

Démarrage du serveur :

godoc -http=:8080 -play

Obtenir de l’aide :

godoc help

Formater le code

Go est un langage strictement typé. Il vous indique comment écrire du code propre et s’occupe du formatage.

Pour formater le dossier racine ou l’endroit où le terminal est ouvert :

go fmt

go fmt .

go fmt ./

Formater tous les dossiers de manière récursive :

go fmt ./...

Compiler avec go build

L’outil de ligne de commande go build prend un fichier .go et génère un fichier exécutable pour Windows et des fichiers binaires pour Linux ou Mac. La création d’un fichier exécutable ou binaire se fera à condition que le fichier .go contient le package main et la fonction principale main.

go build main.go

Exécuter l’application :

./main

# ou

./main.exe

Si vous ne nommez pas votre fichier exécutable ou binaire lors de la construction, Go utilisera le nom du fichier principal. Vous pouvez spécifier un nom avec l’option -o :

go build -o app_name main.go

Compiler avec go run

Cela devient fastidieux lorsque vous modifier fréquemment votre code et que vous devez créer, exécuter et supprimer l’exécutable. Heureusement Go a dans ses outils la commande go run qui va nous aider à effectuer ces trois actions automatiquement en une seule fois.

go run main.go

Compiler avec go install

La différence avec go build c’est que l’exécutable ou le binaire généré est placé dans le répertoire bin de votre espace de travail.

go install hello.go

go
├── bin
│   └── hello
└── src
    └── hello.go

La commande go install fonctionne si le chemin d’accès est défini dans la variable d’environnement GOBIN.

Exemple :

# Afficher la valeur de la variable
go env GOBIN

# Modifier la valeur de la variable
go env -w GOBIN="/home/go/bin"

L’idéal étant de rendre votre répertoire bin global pour pouvoir exécuter vos applications depuis n’importe quel emplacement.

Exemple (.bashrc):

PATH=$PATH:/home/go/bin
