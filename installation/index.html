Installation

  • Répertoires d’installation
  • Emplacement du code
  • Organisation d’un projet

Go est compatible avec les systèmes d’exploitation comme Mac, Linux et Windows. L’installation peut se faire via un installateur automatique, un binaire exécutable à configurer et si votre machine est spécifique il est toujours possible de compiler Go depuis les sources. Vous pouvez visualiser la page des dernières constructions sur Go Build Coordinator.

  • Télécharger Go

Répertoires d’installation

L’installation de Go comprend un répertoire réservé pour la bibliothèque standard et ses binaires et prévoit un espace de travail pour l’utilisateur. Selon la configuration, les chemins sont définis dans les variables d’environnement GOROOT et GOPATH.

Dans un terminal tapez la commande suivante :

go env GOROOT GOPATH

Emplacement du code

L’emplacement du code n’a pas d’importance. Une application peut être compilée dans le répertoire de votre choix. Cependant, si vous souhaitez créer ou partager des packages entre vos différents projets il est préférable que votre code soit placé dans un espace de travail (workspace).

Bien qu’il soit facile d’utiliser un espace de travail, Go intègre un système de modules depuis sa version 1.14 disponible pour la production et apporte beaucoup de souplesse d’une part, pour l’emplacement des bibliothèques qui ne nécessite plus d’avoir un espace de travail et d’autre part améliore grandement le gestionnaire de dépendances permettant de traiter les différentes versions indépendamment pour chaque projet.

Organisation d’un projet

Go n’impose pas de standards officiels mais l’expérience des développeurs a fait émerger des conventions sur la nomenclature et les modèles d’architecture.

Vous trouverez sur github le dépôt golang-standards/project-layout qui propose un ensemble de modèles d’architectures avec une description pour chacun d’entre eux.

Une section dédiée aux modules sera abordé plus loin.
