Identifiant blanc _

Dans Go, Il est possible qu’une fonction retourne plusieurs valeurs et qu’une de ces valeurs ne soit pas utile pour notre programme. Par exemple la fonction Println du package fmt retourne deux valeurs. La première est le nombre d’octets écrits sur la sortie standard et la deuxième toute erreur d’écriture rencontrée.

Nous voulons afficher le nombre d’octets écris mais nous ne souhaitons pas utiliser la variable d’erreur seulement Go stop la compilation pour les variables définies dans une fonction qui sont inutilisées.

package main

import "fmt"

func main() {
    n, err := fmt.Println("Hello, World")

    fmt.Println(n)
}

Sortie :

go run main.go
# command-line-arguments
./main.go:6:5: err declared but not used

Pour éviter ce type d’erreurs, il suffit de remplacer le nom des variables inutilisés par le trait de soulignement _ :

n, _ := fmt.Println("Hello, World")

Nous verrons plus loin que cela fonctionne dans d’autres situations.
