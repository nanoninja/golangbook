Goroutines

  • Utiliser plusieurs Goroutines
  • Attendre la fin des goroutines avec WaitGroup
  • Obtenir des informations sur les goroutines
  • Partager la mémoire avec les goroutines : mutex

Go est livré avec une API de concurrence qui permet d’effectuer des tâches asynchrones via des goroutines. Comparativement à d’autres langages les goroutines n’exploitent pas le mécanisme de threads géré par le système d’exploitation mais intègre son propre gestionnaire.

Les threads (fils d’exécution) sont gérés par le système d’exploitation. Il alloue la mémoire et fait la bascule entre les différents fils d’exécutions en concurrence.

Les goroutines sont gérées par le runtime de Go, elles sont comme des threads en plus léger. Par exemple on pourrait exécuter 1 millions de goroutines pour 1000 threads. Lorsqu’un programme Go s’exécute il embarque la mémoire nécessaire, un garbage collector et son gestionnaire de goroutines.

Pour exécuter une tâche asynchrone dans Go il suffit juste d’ajouter le mot-clé go devant une fonction et c’est tout.

go func () {
    // in progress
}()

Exemple :

package main

import (
    "fmt"
    "time"
)

func worker() {
    time.Sleep(time.Second * 2)

    fmt.Println("done")
}

func main() {
    go worker()
}

Si vous avez testé le code ci-dessus, vous constaterez que rien ne c’est s’affiché dans le terminal car la fonction main est-elle même une goroutine et une fois terminée détruit toutes les autres goroutines.

Dans quelques exemples, nous utiliserons le package time pour faire attendre le programme le temps de faire nos tests.

package main

import (
    "fmt"
    "time"
)

func worker() {
    time.Sleep(time.Second * 1)

    fmt.Println("done")
}

func main() {
    go worker()

    time.Sleep(time.Second * 2)
}

Sortie :

done

Utiliser plusieurs Goroutines

Dans l’exemple suivant nous allons mettre en concurrence plusieurs goroutines avec pour chacune d’entre elle un temps d’exécution plus ou moins long.

package main

import (
    "fmt"
    "time"
)

func worker(name string, d time.Duration) {
    time.Sleep(time.Millisecond * d)
    fmt.Println(name + " is done")
}

func main() {
    go worker("a", 500)
    go worker("b", 1500)
    go worker("c", 300)

    time.Sleep(time.Second * 3)
}

Sortie :

c is done
a is done
b is done

Attendre la fin des goroutines avec WaitGroup

L’utilisation de la fonction de mise en sommeil Sleep est pratique pour simuler le temps d’exécution et réaliser des tests mais cela ne nous indique pas quand est-ce que chaque tâche sera terminée pour quitter le programme proprement.

Dans l’exemple suivant nous exécuterons des goroutines en concurrence avec un temps d’exécution aléatoire pour rendre le teste plus réaliste. Nous ne pourrons pas connaître le temps d’exécution des goroutines à l’avance.

Le package sync avec le type WaitGroup permet de faire attendre le programme principale jusqu’a ce que chaque goroutine signale qu’elle a terminé sa tâche.

package main

import (
    "fmt"
    "math/rand"
    "sync"
    "time"
)

func init() {
    rand.Seed(time.Now().UnixNano())
}

func worker(name string, wg *sync.WaitGroup) {
    defer wg.Done()

    duration := time.Duration(rand.Intn(2000))
    time.Sleep(time.Millisecond * duration)

    fmt.Printf("%s working %d second \n", name, duration)
}

func main() {
    wg := new(sync.WaitGroup)
    wg.Add(3)

    go worker("a", wg)
    go worker("b", wg)
    go worker("c", wg)

    wg.Wait()
}

Obtenir des informations sur les goroutines

package main

import (
    "fmt"
    "math/rand"
    "runtime"
    "sync"
    "time"
)

func init() {
    rand.Seed(time.Now().UnixNano())
}

func worker(name string, wg *sync.WaitGroup) {
    defer wg.Done()

    duration := time.Duration(rand.Intn(2000))
    time.Sleep(time.Millisecond * duration)

    fmt.Printf("%s working %d second \n", name, duration)
}

func info() {
    fmt.Println("CPUs\t\t", runtime.NumCPU())
    fmt.Println("Max CPUs\t", runtime.GOMAXPROCS(0))
    fmt.Println("Goroutines\t", runtime.NumGoroutine())
    fmt.Println("")
}

func main() {
    runtime.GOMAXPROCS(2)
    info()

    wg := new(sync.WaitGroup)

    wg.Add(3)

    go worker("a", wg)
    go worker("b", wg)
    go worker("c", wg)

    info()
    wg.Wait()

}

Partager la mémoire avec les goroutines : mutex

Les goroutines peuvent s’exécuter par millier avec une grande rapidité d’exécution mais comment partager des données entre elles ? Si nous avons une variable counter en mémoire comment les goroutines vont-elles pouvoir lire et écrire de façon concurrente sans qu’il est de conflits ?

package main

import (
    "fmt"
    "sync"
)

var wg sync.WaitGroup
var counter int = 0

func worker() {
    defer wg.Done()
    counter++
}

func main() {

    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go worker()
    }

    wg.Wait()
    fmt.Println("Count: ", counter)
}

Sortie :

Count:  946

Pour detecter les race conditions il faut ajouter l’otption -race au commandes :

go build -race
go run -race
go test -race

Comme on peut le voir nous avons une seule variable en mémoire pour des goroutines qui font la course pour y accéder ce qui nous donne un résultat non statisfaisant pour notre programme.

Pour régler ce problème nous allons utiliser un vérrou qui bloquera une variable en mémoire le temps de lecture et ou d’écrirture puis nous libérerons la mémoire de suite pour que d’autres goroutines puissent y accéder.

package main

import (
    "fmt"
    "sync"
)

var wg sync.WaitGroup
var mu sync.Mutex
var counter int = 0

func worker() {
    defer wg.Done()

    mu.Lock()
    counter++
    mu.Unlock()
}

func main() {

    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go worker()
    }

    wg.Wait()
    fmt.Println("Count: ", counter)
}

Sortie :

go run -race main.go
Count:  1000
