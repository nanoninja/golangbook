Defer

  • Defer multiple
  • Defer avec erreur
  • Defer et renvoie d’erreur
  • Defer avec un fichier

L’instruction defer permet d’exécuter un code même en cas erreur. Il est possible lorsque qu’on obtient une erreur durant l’exécution du code que l’on souhaite effectuer des traitements avant de renvoyer cette dernière. C’est le cas quand on manipule un fichier et qu’une erreur survient de pouvoir libérer la ressource en fermant proprement le fichier.

package main

import "fmt"

func main() {
    defer func() {
        fmt.Println("After main")
    }()

    fmt.Println("Program is running")
}

Sortie :

Program is running
After main

Defer multiple

L’instruction defer peut être utilisée plusieurs fois et l’exécution se fera dans l’ordre : dernier ajouté premier sortie (LIFO).

package main

import "fmt"

func main() {
    defer func() {
        fmt.Println("After main 1")
    }()
    defer func() {
        fmt.Println("After main 2")
    }()
    fmt.Println("Program is running")
}

Sortie :

Program is running
After main 2
After main 1

Defer avec erreur

Si la fonction que vous utilisez avec defer renvoie une erreur celle-ci sera ignorée. C’est au programmeur de la traiter.

package main

import "fmt"

func cleanup(name string) error {
    fmt.Println(name, ":", "running cleanup...")

    return fmt.Errorf("error on cleanup")
}

func main() {
    defer cleanup("cleaner")

    fmt.Println("Program is running")
}

Sortie :

Program is running
cleaner : running cleanup...

Pour gérer l’erreur avec l’instruction defer on enferme son traitement dans une fonction anonyme.

package main

import "fmt"

func cleanup(name string) error {
    fmt.Println(name, ":", "running cleanup...")

    return fmt.Errorf("error on cleanup")
}

func main() {
    defer func() {
        if err := cleanup("cleaner"); err != nil {
            fmt.Println(err)
        }
    }()

    fmt.Println("Program is running")
}

Sortie :

Program is running
cleaner : running cleanup...
error on cleanup

Defer et renvoie d’erreur

Dans certain cas il est nécessaire de retourner une erreur depuis une instruction defer. Par exemple les opérations sur un fichier se sont bien déroulées mais une erreur survient au moment de sa fermeture dans la fonction exécutée par l’instruction defer. Pour récupérer l’erreur on exploite le principe des closures (fermetures) pour alimenter la variable err créé au niveau supérieur.

package main

import "fmt"

func cleanup(name string) error {
    fmt.Println(name, ":", "running cleanup...")

    return fmt.Errorf("error on cleanup")
}

func do() (err error) {
    defer func() {
        err = cleanup("cleaner")
    }()
    return
}

func main() {
    err := do()

    fmt.Println(err)
}

cleaner : running cleanup...
error on cleanup

Defer avec un fichier

package main

import (
    "fmt"
    "log"
    "os"
)

func ReadFile(filename string) {
    f, err := os.OpenFile(filename, os.O_RDONLY, 0600)

    defer func() {
        if err := f.Close(); err != nil {
            log.Println(err)
        }
    }()

    if err != nil {
        log.Fatalln(err)
    }

    i, err := f.Stat()
    if err != nil {
        log.Fatalln(err)
    }

    var b []byte = make([]byte, i.Size())

    if _, err := f.Read(b); err != nil {
        log.Fatalln(err)
    }

    fmt.Println(string(b))
}

func main() {
    ReadFile("./filename.txt")
}
