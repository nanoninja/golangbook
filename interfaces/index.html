Interfaces

Dans le contexte objet, une interface est un type abstrait qui définit le comportement commun des objets à manipuler de manière uniforme.

Un oiseau, un avion ou Superman n’ont pas grand-chose en commun et pourtant ils volent tous. On voudrait faire voler des objets volant sans se soucier de quel type d’objet il s’agit concrètement.

Un oiseau, un avion ou Superman n’ont pas grand-chose en commun et pourtant ils volent tous les deux. On voudrait faire voler mes objets volant sans se soucier de connaître le type concrêt de chacun d’entres eux.

Imaginons un ordinateur avec un corbeille. Cette poubelle pour supprimer n’importe quel objet qui n’ont rien avoir les uns avec les autres et pourtant ils doivent tous avoir le même comportement pour être supprimer.

Les objets sont un fichier texte, une image, une base de données,

package main

import "fmt"

// interface qui sera le point commun de nos objets.
type Deleter interface {
    Delete()
}

// notre poubelle qui prend en paramètre un objet de type Deleter.
func Trash(d Deleter) {
    d.Delete()
}

// type image qui implément l'interface Deleter
type Image struct{}

func (i Image) Delete() {
    fmt.Println("image has been deleted")
}

// type Database qui implément l'interface Deleter
type Database struct{}

func (d Database) Delete() {
    fmt.Println("database has been delete")
}

func main() {

    Trash(Image{})
    Trash(Database{})
}

Sortie :

image has been deleted
database has been delete

Mes objets n’ont à la base rien en commum mais nous pouvons les manipuler de la même manière.
